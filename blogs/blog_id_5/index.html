CoreTuff v1.0 Nightly-1

Overview

  • First public development build of CoreTuff
  • Core systems implemented and under active development

Major Features

  • Teleportation system including /tpask, /tpaccept, /tpdeny, and /tphere
  • Random Teleport (/rtp) with safe location detection and biome filtering
  • Back system (/back) to return to previous locations
  • Ignore system to block teleport requests

EULA System

  • Added EULA enforcement system
  • Plugin now generates eula.txt on first startup
  • Plugin will not enable until user sets accepted: true
  • Prevents usage without agreeing to license terms

Teleport System

  • Functional request-based teleport system
  • Basic validation and request handling implemented
  • Ongoing improvements for stability and edge cases

RTP (Random Teleport)

  • Safe teleportation avoiding unsafe blocks
  • Basic biome filtering
  • Retry system for valid location selection
  • Initial cooldown implementation

Back System

  • Tracks previous player location
  • Works with teleport events and basic scenarios
  • Still being refined for edge cases

Cooldown System

Separate cooldown handling for:

  • /back
  • /rtp
  • /tpask

Config-based values introduced

Ignore System

Commands:

  • /tpignore (ignore teleport requests)
  • /tpuignore (unignore teleport requests)
  • Database-backed storage

Initial caching system implemented.

Folia and Paper Support

  • Compatible with both Paper and Folia
  • Uses thread-safe execution handling
  • Initial implementation of region-safe scheduling

Configuration

  • Basic config.yml support added
  • Includes cooldowns and message settings
  • Configuration system still evolving

Database

  • H2 database integration
  • Used for ignore system persistence
