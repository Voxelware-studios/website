CoreTuff v1.0 Nightly-2

Overview

  • Second public development build of CoreTuff

Major Features

  • Added Teleport countdown (For all teleportation oriented commands).
  • Renamed /hunger command to /feed.
  • Changed all permission keys.
  • Added player field for these commands:
    • /gmc
    • /gma
    • /gms
    • /gmsp
    • /heal
    • /feed
  • Integrated Placeholder API with CoreTuff.
  • Added prefix for plugin message. Can be configured in config.yml.
  • Cooldown message added in config.yml.
  • Updated from Paper api 1.21 to 26.1
  • Updated from Java 21 to 25.

EULA System

  • Changed text generated in eula.txt.

New Permissions

  • Gamemode command permissions:
    • coretuff.gamemode.gma
    • coretuff.gamemode.gma.others
    • coretuff.gamemode.gmc
    • coretuff.gamemode.gmc.others
    • coretuff.gamemode.gmsp
    • coretuff.gamemode.gmsp.others
    • coretuff.gamemode.gms
    • coretuff.gamemode.gms.others
  • Utility command permissions:
    • coretuff.utility.fly
    • coretuff.utility.heal
    • coretuff.utility.heal.others
    • coretuff.utility.feed
    • coretuff.utility.feed.others
  • Teleportation:
    • coretuff.teleportation.back
    • coretuff.teleportation.tpask
    • coretuff.teleportation.tpaccept
    • coretuff.teleportation.tpdeny
    • coretuff.teleportation.tphere
    • coretuff.teleportation.rtp
    • coretuff.teleportation.tpignore
    • coretuff.teleportation.tpunignore

Known Issues

  • Possible edge-case bugs in teleport handling
  • Some systems may require further optimization

Notes

  • To use the command with player field you need .others command.
  • Currently you can use %player_name% Placeholder API's placeholder. Usage is mentioned in config.yml.
  • If you currently use Nightly 1 we recommend you to delete config.yml and eula.txt to regenerate it.
  • DO NOT DELETE database.mv.db you will use ignore data.S
  • This is a nightly development build, It might be unstable
  • You can report any bugs in our Discord
